** 赛博书桌
现在我使用的是 [[https://github.com/miniflux/v2][GitHub - miniflux/v2: Minimalist and opinionated feed reader]] 这个阅读器，因为它提供了
全文抓取和图片代理的功能，基于 Go 语言编写性能和部署容易度都有保障。不过在阅读方式上还是有点麻烦因为是基于
模板语法直接渲染成 HTML 返回的，而没有做单页应用。所以在阅读条目和返回未读都是一个新的页面，在网络环境不佳
的时候还是比较折磨的。之前有试过一些通过 API 渲染的前端项目例如 [[https://github.com/electh/ReactFlux][GitHub - electh/ReactFlux: A Simple but Powerful RSS Reader for Miniflux]] ，
意外之喜是 Miniflux 的 API 会返回 HTML 内容，所以一次请求可以批量拉取条目而不必再分成多个 HTML 页面，但是由他人做的前端应用设计总有这里或者那里的小「毛刺」，在更多地阅读未读条目后也开始越来越不想在一个前端应用去看了。

所以还有什么方式呢？最后果然还是向 =Emacs= 去寻找了，去网上找了有没有相关实现发现确实有人做了不过不怎么完善，
[[https://sr.ht/~rprospero/miniflux.el/][miniflux.el: Read RSS feeds from your Miniflux server in Emacs]] ，在这周也是花了时间去完善最后感慨早
该这样了。因为只要将文字接入 =Emacs= 就能自动接入其它为文字处理流程所做的成果，例如翻译、代办、搜索什么的。这就有点
像是「书桌」，为其布置了很多东西，阅读灯、香薰、置物架等等等等，为其花那么多努力这样之后做阅读写作的事也更希望在
这个「书桌」上做了。

甚至在越看挺多 Hacker News 源之后我发现看一篇文章还是挺想看一下这篇文章的在 HN 热度的。在浏览器我还得自己
新建标签页打开 HN ，但在 =Emacs= 里糊点代码就实现自动抓取热度数据了。
